Business Problem
01
The system no longer matched the organisation.

YourTribe’s internal hiring operations were managed through an admin panel originally built for a much smaller version of the business.

As workflows expanded, roles evolved, and operational complexity increased, the platform became structurally misaligned with how teams actually worked.

Users increasingly relied on tribal knowledge, manual coordination, and experience-based navigation instead of system guidance.

The challenge wasn’t improving isolated screens — it was rebuilding the operational foundation before complexity became the ceiling on scale.

North Star
Workflow Efficiency

Reduce steps required to complete core hiring workflows across sourcing, screening, validation, and client sharing.

Guardrail
Validation Integrity

Improve operational speed without removing the accountability and approval layers critical for hiring quality.

Supporting Metrics
Operational Clarity

Reduce role confusion, cross-module switching, onboarding complexity, and dependency on experienced users.

The Reframe
The problem wasn’t poor usability on individual screens.

The system encoded an organisation that no longer existed.
Design Principles
02
01
Encode the org

Ownership, permissions, and responsibilities should live inside the system — not inside people's heads.

02
Structure the workflow

Users should follow predictable stages instead of relying on memory and operational experience.

03
One connected lifecycle

Companies, jobs, and candidates should exist inside one continuous operational system.

04
Preserve accountability

Speed matters, but not at the cost of validation integrity and hiring quality.

Solution Design
03
01

Role-based visibility & control

I redesigned the platform around role-based visibility, permissions, and ownership so users only see workflows relevant to them.

  • Role-based navigation & permissions
  • Personalized module visibility
  • Maker–checker approval flow
  • Admin-controlled role configuration
  • Restricted access based on responsibilities
[ Role hierarchy / permissions system visual ]
[ Workflow stage simplification visual ]
02

Task & workflow simplification

The hiring process was transformed into a clear stage-based operational flow: Sourcing → Screening → Validation → Client Sharing.

  • Reduced redundant workflow loops
  • System-triggered notifications
  • Standardized hiring progression
  • Lower learning curve for new users
  • Reduced dependency on experienced team members
03

Unified architecture & lifecycle

The platform was reorganized around connected relationships between companies, jobs, and candidates — turning fragmented modules into one operational lifecycle.

  • Connected hiring lifecycle
  • Centralized candidate visibility
  • Reduced module switching
  • Separated candidate & user data layers
  • Improved scalability and data consistency
